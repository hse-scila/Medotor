## 📋 Описание проекта

**LM Studio Project** — это полнофункциональная AI-система для медицинской диагностики ЛОР-заболеваний, построенная на базе современных технологий машинного обучения и обработки естественного языка. Проект объединяет языковые модели, системы поиска и генерации (RAG/MemoRAG), управление данными пациентов и обработку медицинских документов.

### Ключевые возможности

- 🧠 **Языковые модели** - интеграция с Ollama для локальных моделей
- 📚 **RAG и MemoRAG системы** - интеллектуальный поиск в медицинской литературе
- 👥 **Управление пациентами** - база данных с документами и историей
- 🔍 **OCR функциональность** - распознавание текста из медицинских изображений
- 📄 **Обработка документов** - поддержка PDF, DOCX, изображений
- 🌐 **Веб-интерфейс** - современный UI с темной/светлой темой
- ⚡ **Real-time обновления** - WebSocket для отслеживания прогресса

## 🚀 Быстрый старт

### Простое решение

1. **Запустите проект:**
   ```bash
   start.bat
   ```
   Это автоматически:
   - Проверит и освободит порт 8000 (если занят)
   - Запустит backend сервер
   - Откроет браузер с веб-интерфейсом

2. **Откройте в браузере:**
   ```
   http://localhost:8000
   ```

3. **Остановка сервера:**
   ```bash
   stop.bat
   ```
   или закройте окно терминала с сервером

### Требования

- **Python** 3.8+
- **Anaconda** с окружением TorchLama (для GPU оптимизации)
- **Ollama** - локальные языковые модели
- **Tesseract OCR** (опционально, для OCR функциональности)

### Установка зависимостей

```bash
pip install -r requirements.txt
```

## 📡 API Эндпоинты

### Система
- `GET /` - веб-интерфейс
- `GET /api` - информация об API
- `GET /health` - проверка состояния

### Модели
- `GET /v1/models` - список моделей
- `POST /v1/chat/completions` - генерация (OpenAI-совместимый)
- `POST /load-model` - загрузка модели
- `GET /available-models` - доступные модели

### Ollama
- `POST /ollama/connect` - подключение к Ollama
- `GET /ollama/models` - модели Ollama
- `POST /ollama/generate` - генерация через Ollama
- `POST /ollama/generate-stream` - стриминг генерации

### RAG System
- `POST /rag/upload-file` - загрузка файла
- `POST /rag/documents` - добавление документов
- `POST /rag/search` - поиск в RAG
- `GET /rag/stats` - статистика RAG
- `POST /rag/chat` - чат с RAG
- `POST /rag/chat-stream` - стриминг чата

### MemoRAG System
- `POST /memorag/documents` - добавление документов
- `POST /memorag/search` - поиск с памятью
- `POST /memorag/chat` - чат с MemoRAG
- `GET /memorag/memory-stats` - статистика памяти

### Пациенты
- `GET /patients` - список пациентов
- `POST /patients` - создание пациента
- `GET /patients/{id}` - информация о пациенте
- `PUT /patients/{id}` - обновление пациента
- `DELETE /patients/{id}` - удаление пациента
- `POST /patients/{id}/documents` - добавление документа
- `GET /patients/{id}/documents` - документы пациента

### Выгрузка базы данных пациентов
- `GET /patients-db/export?format=sqlite` - экспорт БД в виде SQLite файла (`.db`), полная копия базы
- `GET /patients-db/export?format=sql` - экспорт БД в виде SQL дампа (`.sql`) с INSERT для всех таблиц
- `GET /patients-db/export?format=json` - экспорт БД в JSON (`.json`)
  - Поля: `export_info`, `statistics`, `patients`, `documents`, `patients_with_documents`
  - `patients_with_documents` — удобная структура: каждый пациент содержит массив своих документов (`documents`)
- `GET /patients-db/export?format=zip` - экспорт в ZIP архив (`.zip`), содержащий `.db`, `.sql` и `.json` с `patients_with_documents`

### Импорт базы данных пациентов
- `POST /patients-db/import` - импорт БД из внешнего файла
  - **Параметры:**
    - `file` (FormData) - файл для импорта (`.db`, `.sql` или `.json`)
    - `mode` (FormData) - режим импорта: `merge` (добавить к существующим) или `replace` (заменить все)
  - **Поддерживаемые форматы:**
    - **SQLite (`.db`)** - полная замена базы данных
      - Доступен только режим `replace`
      - Автоматически создается бэкап существующей БД перед заменой
      - Проверяется валидность импортируемой БД
      - При ошибке выполняется автоматическое восстановление из бэкапа
    - **SQL (`.sql`)** - выполнение SQL скрипта
      - Поддерживаются режимы `merge` и `replace`
      - SQL файл должен содержать `BEGIN TRANSACTION`
      - При ошибке выполняется откат транзакции
    - **JSON (`.json`)** - импорт из структурированных данных
      - Поддерживаются режимы `merge` и `replace`
      - Поддерживается структура `patients_with_documents` (каждый пациент с массивом документов)
      - Поддерживается старый формат (`patients` + `documents` отдельно)
      - Автоматическое связывание документов с пациентами
  - **Особенности:**
    - Если база данных не существует, она создается автоматически с правильной структурой
    - Все связи между пациентами и документами сохраняются
    - После успешного импорта автоматически обновляется статистика и список пациентов

### OCR
- `POST /ocr/extract-text` - извлечение текста
- `GET /ocr/status` - статус OCR

### Конфигурация
- `GET /config` - краткая конфигурация
- `GET /config/full` - полная конфигурация
- `POST /config/reload` - перезагрузка конфигурации

## 🧠 Ключевые компоненты

### 1. RAG System
Система поиска и генерации на основе векторных хранилищ:
- Поддержка FAISS и ChromaDB
- Автоматическое разбиение документов на чанки
- Поиск похожих документов
- Эмбеддинги через Sentence Transformers или Ollama

### 2. MemoRAG System
Продвинутая система с глобальной памятью:
- Глобальная память для всей системы
- Двухэтапная обработка запросов
- Генерация подсказок для ретривера
- Компрессия памяти для оптимизации

### 3. Patients Database
SQLite база данных для управления пациентами:
- CRUD операции
- Управление документами
- Автоматическая миграция данных
- Статистика и отчеты

### 4. File Processor
Обработка различных форматов документов:
- PDF (PyPDF2)
- DOCX (python-docx)
- Изображения (JPEG, PNG, BMP, TIFF)
- Текст (TXT)

### 5. OCR Module
Распознавание текста из изображений:
- Tesseract OCR
- Поддержка русского и английского языков
- Предобработка изображений
- Интеграция с массовым импортом

## 📖 Использование

### Запуск сервера

```bash
# Полный запуск (backend + frontend)
start.bat

# Только backend
start_server.bat

# Только frontend (если backend уже запущен)
start_frontend.bat
```

### Работа с пациентами

1. Откройте веб-интерфейс: `http://localhost:8000`
2. Перейдите в раздел "Пациенты"
3. Добавьте нового пациента или выберите существующего
4. Загрузите документы пациента
5. Используйте чат для анализа данных пациента

### Работа с RAG

1. Загрузите медицинские документы (PDF, DOCX)
2. Документы автоматически индексируются
3. Используйте поиск для нахождения релевантной информации
4. Используйте чат с RAG для получения ответов на основе документов

### OCR функциональность

1. Загрузите изображение с текстом
2. Система автоматически распознает текст
3. Результаты можно использовать в чате или сохранить

### Экспорт и импорт базы данных пациентов

#### Экспорт базы данных

1. Откройте раздел "Пациенты" в веб-интерфейсе
2. В секции "Database Export/Import" выберите формат экспорта:
   - **SQLite (.db)** - полная копия базы данных
   - **SQL (.sql)** - SQL скрипт с данными
   - **JSON (.json)** - структурированные данные с удобной структурой `patients_with_documents`
   - **ZIP Archive** - все форматы в одном архиве
3. Нажмите "Export Database"
4. Файл будет сохранен в папку "Загрузки" (Downloads) вашего браузера

#### Импорт базы данных

1. Откройте раздел "Пациенты" в веб-интерфейсе
2. В секции "Database Export/Import" выберите режим импорта:
   - **Merge** - добавить данные к существующим
   - **Replace** - заменить все данные (⚠️ требует подтверждения)
3. Нажмите "Select File and Import"
4. Выберите файл для импорта (`.db`, `.sql` или `.json`)
5. При режиме `replace` подтвердите операцию
6. После успешного импорта данные автоматически обновятся

**Важно:**
- При импорте SQLite файла в режиме `replace` автоматически создается бэкап текущей БД
- Если база данных была удалена, она автоматически создается при импорте
- Все форматы поддерживают восстановление полной структуры данных

## 🔍 Устранение неполадок

### Проблема: Не удается подключиться к Ollama

**Решение:**
1. Убедитесь, что Ollama запущен: `ollama serve`
2. Проверьте URL в конфигурации: `http://127.0.0.1:11434`
3. Перезапустите backend сервер

Подробнее см. [INSTRUKCIYA_PODKLYUCHENIYA.md](INSTRUKCIYA_PODKLYUCHENIYA.md)

### Проблема: Порт 8000 занят

**Решение:**
```bash
# Остановите старый процесс
stop.bat

# Или вручную найдите и завершите процесс
netstat -ano | findstr :8000
taskkill /PID <PID> /F
```

### Проблема: Ошибки при обработке PDF

**Решение:**
1. Убедитесь, что файл не поврежден
2. Проверьте размер файла (максимум 50MB по умолчанию)
3. Проверьте логи: `backend/logs/api_server.log`
